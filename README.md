# 项目指南针 · Project Compass

> 一个帮助个人或小团队在高不确定性场景下做出"边界清晰的投入决策"的判断辅助工具。

---

## 这个工具对谁有价值

**主要适用人群：**

- **BD / 业务拓展人员**：面对大量外部机会（RFP、合作邀约、新市场方向），需要快速判断值不值得花时间深入。
- **早期产品负责人或创始人**：在资源有限的情况下，需要对多个潜在方向做"值不值得押注"的判断。
- **技术负责人 / 解决方案架构师**：收到潜在客户需求或合作提案，需要在正式评估前先做可行性预判。
- **个人投资决策者**：面对信息不完整的机会，想要系统整理已知事实、主张和未知项，再决定是否推进。

**核心价值：**

1. **把"感觉"变成"结构"**：不是拍脑袋说"感觉不靠谱"，而是把公开事实、发起方主张、当前未知三类信息分层放好，清楚看到自己在什么基础上做判断。
2. **逼出真正的风险点**：通过五个维度（商业化、用户、技术、适配、风险）逐一问"这件事成立需要相信什么"，把隐藏的悖论信号和红旗显性化。
3. **限定投入边界，而不是给出"投还是不投"**：决策快照的结果是一个承诺边界（暂不继续 / 继续观察 / 可以接触 / 可以投入有限试点），明确当前愿意付出多少，而不是非黑即白。
4. **版本化记录，支持复盘**：每次出现新信息都可以新增一个决策快照，历史版本保留；最终支持复盘归档，记录判断是否出了问题、为什么出了问题。

## 功能模块与使用流程

```
导入机会 → 搜集信息分层 → 评估五维假设 → 冻结决策快照 → 持续跟踪 → 复盘归档
```

### 第一步：导入机会

进入「后台管理 → 机会导入」，填写：
- 机会来源链接（URL）
- 机会名称
- 类别（如：提案需求 / AI、安全问卷 AI 等）
- 简短描述

同时可在「决策方现状」中填写自身背景（角色、目标、资源、风险承受度、最大投入上限），帮助后续判断时锚定自己的立场。

---

### 第二步：信息搜集与分层（Tab A）

进入某个机会的分析页面，第一个标签页展示三列：

| 列名 | 含义 |
|------|------|
| 公开事实 | 有来源、可核验的信息（官网定价、集成列表、公开案例等） |
| 发起方主张 | 对方声称但未独立验证的信息（"节省 90% 时间"等）|
| 当前未知 | 尚未找到答案的关键问题 |

每条信息可以：
- 通过下拉框手动调整**层级**（公开事实 / 发起方主张 / 当前未知）
- 通过下拉框手动调整**可信度**（高 / 中 / 低）
- 点击来源链接跳转原始页面

---

### 第三步：假设·证据·红旗（Tab B）

系统预设五个判断维度，每个维度展开后显示：

- **乐观前提**：这件事成立需要相信什么
- **当前证据摘要**：支撑 / 削弱这个前提的信息
- **悖论信号**：已知信息中与前提矛盾的部分
- **关联红旗**：潜在的致命问题，每条红旗可定级为「致命 / 需关注 / 已解释」

完成所有维度评估后，对每个维度给出整体判断：**支撑 / 存疑 / 否定**。

---

### 第四步：决策快照（Tab C）

所有维度评估完毕后，可生成一个**决策快照**，包括：

- 当前的**承诺边界**（愿意投入多少）
- 各维度判断汇总
- 升级 / 暂停 / 退出的触发条件
- 下次复核日期

**多版本机制**：若后续有新信息进来（信息搜集页会标记"有新信息"），可新增一个快照，旧版本自动归档、可展开对比。

对于当前**存疑或否定**的维度，可在快照中直接备注"需跟踪什么信号"，不再需要跳转到单独的跟踪页。

---

### 第五步：复盘归档（Tab D）

机会结束（无论是推进还是放弃）后，在此记录：

- 边界是否发生了变化
- 判断是否出现了错误
- 如果出现错判，是什么原因导致的（需手动填写，不由 AI 代劳）

历史复盘记录按时间顺序保留，支持团队沉淀判断经验。

## 技术说明

- **前端**：原生 HTML + CSS + JavaScript（无框架依赖），macOS 风格设计系统
- **后端**：[Hono](https://hono.dev/) 框架，运行于 Cloudflare Workers
- **部署**：Cloudflare Pages
- **存储**：当前版本数据存储在浏览器 `localStorage`（`compass_project_states_v3`），无需后端数据库
